# Architecture

**Idea funnel:** many inputs → filter for signal → the few worth a person's time.

```
GitHub Actions (daily cron, free on public repos)
 └─ python -m engine.run
     1. INGEST     podcasts RSS · Apple top-charts · news RSS · EDGAR S-1/F-1
     2. EXTRACT    Gemini free tier → Groq free tier → rule-based fallback
     3. VALIDATE   SEC company_tickers.json + yfinance  ← anti-hallucination gate
     4. CORROBORATE  Google Trends (optional) · EDGAR full-text mentions
     5. SCORE      multi-source count · chart velocity · novelty vs 30-day history
     6. PUBLISH    docs/index.html (GitHub Pages) · Gmail SMTP digest
     7. COMMIT     data/history/YYYY-MM-DD.json back to the repo (free state)
```

## Why these choices

| Concern | Choice | Why |
|---|---|---|
| Compute | GitHub Actions cron | Free unlimited minutes on public repos; ~2 min/day usage; no server. |
| Hosting | GitHub Pages from `/docs` | Free, regenerated by the same job. |
| State | JSON files committed to repo | Free, versioned, inspectable; no DB to babysit. |
| Email | Gmail SMTP app password | Free, no third-party mailer account to expire. |
| Podcasts | RSS metadata + show notes | Free and durable. Feeds resolved from Apple Podcast IDs via the free iTunes Lookup API, so config stays human-editable and self-healing. |
| App charts | Apple `rss.marketingtools.apple.com` | Official, keyless, JSON. Yesterday's snapshot is stored so *velocity* (rank Δ, new entrants) is computed, not guessed. |
| Filings | SEC EDGAR full-text search (`efts.sec.gov`) | Official + free; 10 req/s cap respected; used both as an ingest source (new S-1/F-1s) and as corroboration (does a trend appear in filings?). |
| Extraction | Gemini Flash free tier (~1,500 req/day cap; we use <10) | Best free quality; Groq free tier as fallback; deterministic rule-based extractor as last resort so the pipeline never dies silently. |
| Ticker truth | SEC `company_tickers.json` (official) + yfinance | The gate: candidates must match the official SEC map, or pass a live listing check (for non-US, e.g. `9992.HK`). Fail → discarded or ticker-less Watchlist. **Never guessed.** |

## What canNOT be done reliably for free (and what we do instead)

1. **Full podcast audio transcription.** Whisper-quality transcription of ~40
   episodes/week has no durable free path (free ASR APIs are trials; YouTube
   auto-caption scraping gets IP-blocked from cloud runners). **Workaround:**
   episode titles + show notes, which for finance podcasts are dense with names
   and theses. Cost of the compromise: we catch *what episodes are about*, not
   offhand mid-episode mentions.
2. **Google Trends.** `pytrends` was archived in April 2025; every free wrapper
   is unofficial and rate-limited. **Workaround:** optional `trendspy` module,
   ≤10 queries/day, any failure → run continues without Trends (it's a score
   bonus, never a dependency).
3. **Google Play charts.** No official feed; scraping breaks randomly.
   **Workaround:** omitted; Apple charts cover the same cultural surface.
   (A Play module can be added later as another best-effort ingestor.)
4. **Reuters RSS.** Discontinued publicly. **Workaround:** Yahoo Finance RSS +
   targeted Google News RSS queries.
5. **Free-LLM longevity.** Gemini/Groq free-tier terms can change without
   notice. **Workaround:** provider chain + rule-based fallback + per-source
   status line on the dashboard, so degradation is visible, never silent.

## Reliability principles

- Any single source failing yields an empty list, not an exception; the run
  only fails (red ✗ in Actions, which emails you) if **all** ingestion fails.
- Every published claim carries its source links; EDGAR/App-chart items link to
  the primary artifact.
- Repeat suppression: a trend already flagged in the last 30 days is only
  re-shown if its score jumps ≥25% ("accelerating").
- Megacap noise filter: routine AAPL/NVDA/etc. coverage is penalized unless it
  shows up across ≥3 independent source types.
